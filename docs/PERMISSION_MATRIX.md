# 权限矩阵

更新时间：2026-06-29

## 角色定义

| 角色         | 标识        | 说明                                       |
| ------------ | ----------- | ------------------------------------------ |
| 学生         | `student`   | 参与项目、提交申请、查看资料、使用 AI      |
| 教授         | `professor` | 管理项目、审批申请、发布会议、查看团队进展 |
| 实验室管理员 | `lab_admin` | 全局管理账号、库存、知识库和全量数据       |

## 权限原则

- `student` 只能处理自己参与范围内的数据
- `professor` 处理自己负责或管理的项目数据
- `lab_admin` 具备全局视图和管理权限
- 项目维度权限通过 `projects.project_member` 继续细分

## 当前前端分级

- 所有角色可见：`Dashboard`、`项目管理`、`耗材管理`、`文件资料`、`会议通知`、`AI 助手`
- `账户管理` 对所有角色可进入，但内容分级：
  - `student`：仅个人资料与安全设置
  - `professor`：个人资料，后续可扩展项目成员视图
  - `lab_admin`：个人资料 + 成员创建 + 角色调整 + 停用管理

## 关联数据模型

项目成员关联表用于细化项目内可见性与职责：

```text
projects.project_member
  (project_id, user_id, member_role, joined_at)
```

典型 `member_role`：

- `leader`
- `member`
- `advisor`
- `manager`
