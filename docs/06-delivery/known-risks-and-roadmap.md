# 已知风险与路线图

适用对象：项目负责人、整合实施方、架构与产品决策者  
更新时间：2026-07-01  
关联文档：[项目进度](./project-progress.md)、[OpenAPI 治理说明](../04-api/openapi-governance.md)

## 1. 当前已知风险

### 1.1 契约与实现存在局部差异

- OpenAPI 未完全覆盖所有新接口
- 共享类型中仍有旧成员角色痕迹
- 认证与共享角色体系仍存在“6 级定义 / 3 级主用”的现实差异

### 1.2 Windows 本地开发兼容性

- 文件监听、依赖解析在部分环境中仍有噪音
- 更推荐 Docker 路径或本机直跑前端

### 1.3 外部集成仍偏轻量

- NAS 仍主要是路径 / 链接登记
- AI 文件入库已支持，但知识检索仍可继续增强

## 2. 路线图

### 近阶段

- 完成文档与代码统一
- 补齐契约与共享类型
- 收敛项目主线的跨模块边界

### 中阶段

- 明确跨组整合接口与模块保留策略
- 强化结构化汇报、项目资料、项目树联动
- 补更完整的自动化测试

### 后阶段

- 统一认证接入
- 正式 NAS 集成
- 更成熟的生产部署与运维方案
